# 文档编写：赋文字以形状，给开发以声音

默认以双换行来表示换行

不要滥用加粗：加粗的目的是强调，如果所有内容都加粗，等于没有强调，但英文文档需要大写 NOT

不要滥用括号：括号会影响编辑和查看时的视觉效果，应善用各类连词和句式来替代

## 视图
- 对于开发者文档和所有的草稿、中间文档，禁用 Markdown 图表，DX 太差
- 总览某目录时，应展示使用制表符的树视图，目录结尾应带 `/`。
- Mermaid 适合展示任务图，但尽量不要用于草案

## 链接与路径
- 善用 `[md链接格式]()` 可以优化文档的 UX 和 DX。
- 文档应使用相对自身的链接，而不是盘上的绝对路径；善用 `../`，但不应超过五层
- 文档层级太深时，在不引起歧义的情况下，以 空、 `/` 或项目名指示根目录，但不应表示为链接

## 列表使用
- 善用无序列表和有序列表，注意它们的语义
- 如果一个列表的叶子节点成句，则该列表不应超过两层；如果是词或短语，允许三层。
- 如果叶子节点结尾不是成句，不要加句号
- 一个列表的根少于两栏，视觉效果很诡异；同理，四个**其中最大换行不超过两行**的句子群不成列表也不合适
